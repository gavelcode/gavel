## Unreleased

### Operations

- **Breaking: first-run bootstrap moves into the schema.** The default
  tenant (`slug=default`) and the first admin user (`admin@local`) are
  now seeded by `seed.sql`, applied as part of `database.Migrate()`. The
  initial admin password is the literal `changeme` and the user is
  flagged `must_change_password=true`, so the first login is forced to
  set a real password before doing anything else.
- The Go function `platform.Bootstrap()`, the `BootstrapDeps` /
  `BootstrapConfig` types, and the `GAVEL_BOOTSTRAP_ADMIN` env var are
  gone. The bootstrap admin email is no longer configurable — it is the
  fixed `admin@local`, which the operator should change immediately
  after first login.
- The Argon2id hash of `changeme` uses a fixed salt so the seed is
  reproducible across rebuilds. This is acceptable because the password
  itself is public; the security control is the
  `must_change_password=true` flag and the assumption that the operator
  logs in before exposing the server.

### API

- **New: yaml-first `/api/v1/` surface.** `core/userinterface/api/v1/openapi.yaml`
  is now the source of truth; Go server stubs and the TypeScript web client
  are regenerated by `make openapi-gen`. The Scalar reference is served at
  `GET /api/v1/docs`, the raw spec at `GET /api/v1/openapi.yaml`.
- **Breaking: `/api/` (v0) is deprecated.** Every v0 response now carries
  `Deprecation: true`, `Sunset: Wed, 31 Dec 2026 23:59:59 GMT`, and a
  `Link` header pointing to the v1 successor and its docs. v0 will be
  removed on the sunset date.
- **Breaking: pagination model.** v1 list endpoints return
  `{ items: [...], next_cursor: string | null }` and accept
  `?limit=&cursor=`. v0 used `{ items, total }` with `?limit=&offset=`.
- **Breaking: JSON casing.** All v1 payloads use `snake_case`. v0 mixed
  camelCase (`pleading`) and snake_case (`project`, `casefile`).
- **Breaking: auth endpoints.**
    - `POST /api/auth/login`           → `POST /api/v1/sessions`
    - `POST /api/auth/logout`          → `DELETE /api/v1/sessions/current`
    - `GET /api/auth/me`               → `GET /api/v1/me`
    - `POST /api/auth/change-password` → `POST /api/v1/me/password`
- **Breaking: token id type.** v1 surfaces API token ids as UUIDs (strings)
  rather than v0's numeric ids.
- **Breaking: token endpoints.**
    - `GET/POST/DELETE /api/tokens`    → `GET/POST/DELETE /api/v1/me/tokens`
- **Breaking: admin endpoints.**
    - `POST /api/users`                → `POST /api/v1/admin/users`
- **Breaking: project sub-resources.** `/api/casefiles?project_id=…` and
  `/api/pleadings?project_id=…` now have first-class sub-resource forms:
    - `GET /api/v1/projects/{key}/casefiles`
    - `GET /api/v1/projects/{key}/pleadings`
    - `POST /api/v1/projects/{key}/pleadings`
    - `GET /api/v1/projects/{key}/baseline`
  The flat endpoints `GET /api/v1/casefiles` and `GET /api/v1/pleadings`
  remain, scoped via `project_id`, `gavelspace`, and `status` query params.
- **New: gavelspace management endpoints.** Creating a gavelspace and
  registering or removing projects used to happen as a side-effect of
  ingestion; v1 exposes them explicitly:
    - `POST /api/v1/gavelspaces` (admin)
    - `POST /api/v1/gavelspaces/{name}/projects` (admin)
    - `DELETE /api/v1/gavelspaces/{name}/projects/{project_id}` (admin)
- **New: project admin endpoints.** `POST /api/v1/projects`,
  `PUT /api/v1/projects/{key}/quality-gate`, `PUT /api/v1/projects/{key}/languages`.
- **Deferred: `POST /api/analyses`.** The monolithic multipart ingest stays
  on v0 until the v1 two-step submit redesign lands. The CLI keeps
  targeting `/api/analyses` for now; `FetchBaseline` already moved to v1.

### Clients

- **Web (`apps/web`)** consumes v1 end-to-end. Generated TypeScript types
  live at `src/shared/api/v1.gen.ts`. Refresh with `pnpm openapi-gen`.
- **CLI (`apps/cli`)** uses v1 for `FetchBaseline`. `SubmitAnalysis` still
  uses v0 pending the ingest redesign.
